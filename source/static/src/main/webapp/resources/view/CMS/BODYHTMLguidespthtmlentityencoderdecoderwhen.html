HTML Entity Encoder & Decoder - Quando Usar

Last reviewed: 2026-07-23

Use o HTML Entity Encoder & Decoder sempre que um texto precisar aparecer corretamente dentro de HTML em vez de ser interpretado como marcação - um campo de comentário, um trecho de código em um post, ou qualquer lugar onde texto digitado pelo usuário entra na página.


Quando faz sentido

Use antes de colar texto digitado pelo usuário ou não confiável em HTML puro, para que caracteres como &amp;, &lt; e &gt; apareçam como texto literal em vez de quebrar a marcação. Também funciona ao contrário: cole um trecho de HTML que já contém entidades e decodifique-o de volta para texto simples.


Quando não faz sentido

Esta ferramenta escapa apenas cinco caracteres - ela não remove tags nem scripts. Se o objetivo é remover marcação perigosa em vez de exibi-la como texto, use um sanitizador de HTML dedicado. Também não é a ferramenta certa para codificar dados binários ou arquivos inteiros para transporte - veja Base64 Encoder & Decoder para isso.


O que esperar

A direção é detectada automaticamente a partir do seu texto, sem alternância manual entre codificar e decodificar. Tudo roda localmente nesta aba do navegador - sem upload, sem conta, e nada do que você digita é salvo depois que você sai da página.
