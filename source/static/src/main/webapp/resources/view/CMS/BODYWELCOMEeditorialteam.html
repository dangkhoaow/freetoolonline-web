Behind every tool page sits a layered review cadence and four editorial principles you can verify on each page you read.


Editorial principles you can verify

  • Browser-first where possible, server-side when needed. Lightweight utilities run in your browser tab; tools that need a native binary upload over HTTPS, run server-side, and the file is removed shortly after.
  • Answer first, depth second. Every page answers its core question at the top, with the trade-offs below.
  • Corrections are appended, not hidden. They are added under the original paragraph and dated rather than silently overwritten.
  • Dated reviews you can verify. The "Last reviewed" date only moves once all three editors have signed off on the change.

Review cadence at a glance

  • Same day: reader-reported issues are triaged the same day they arrive through the contact form.
  • Weekly: one tool category is tested end-to-end every week in current Chrome, Firefox, Safari, and Edge.
  • Monthly: the full catalogue gets a sweep every month.
  • Quarterly: a visual check on every screen size from phone to external monitor runs every quarter.

The dated review credit

The dated review credit at the top of each tool page names the editor who last touched it, so you can see at a glance how recently a page was checked.
