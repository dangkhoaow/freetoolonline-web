How the server-side FFmpeg pipeline handles your file

FFmpeg Online uploads your file over HTTPS to a hosted processing service, dispatches your ffmpeg or ffprobe command to a server-side worker, streams the console log to the page in real time so you can follow encoding progress, and returns the output as a one-click download. The browser holds the upload form and the command builder; the encoding, transmuxing, or metadata probe runs on the service host, so your device is not CPU-bound. For workflows that must keep the source on-device, run a local FFmpeg install instead.


Command patterns that cover most jobs

Format conversion (MP4 → WebM). -i input.mp4 -c:v libvpx-vp9 -b:v 1M -c:a libopus output.webm - re-encodes to VP9 video and Opus audio for modern browsers.

Trim a segment (keep seconds 10-30). -i input.mp4 -ss 10 -t 20 -c copy output.mp4 - -c copy avoids a re-encode, so the trim is instant.

Extract audio only (MP4 → MP3). -i input.mp4 -vn -c:a libmp3lame -b:a 192k output.mp3.

Resize to 720p. -i input.mp4 -vf scale=1280:720 -c:v libx264 -crf 23 output.mp4 - bicubic scale + quality-targeted H.264 re-encode.

Probe a file before editing. Switch the binary to ffprobe and pass -v quiet -print_format json -show_format -show_streams input.mp4 to get codec, bitrate, duration, and stream metadata as JSON.

Remember: the forward-slash is reserved by this tool's command parser, so express paths without leading slashes and avoid the / separator in filter chains.


FFmpeg vs FFprobe - pick the right binary first

The first dropdown at the top of the command builder switches the binary between ffmpeg and ffprobe; pick the right one before you type any parameters because the two binaries answer different questions. FFmpeg is the encoder and transmuxer - use it when you want to re-encode (MP4 to WebM), trim by seconds, resize to 720p, or extract an audio track. FFprobe is the read-only inspector - use it when you want the codec, bitrate, duration, frame rate, and stream list as JSON without producing an output file (handy when you are debugging "why does this file not play?"). The two share the same command-parameters input shape and the same forward-slash restriction, but only FFmpeg produces a downloadable result; FFprobe writes its findings to the console output area.


Privacy and retention

Your file is uploaded over HTTPS to service.us-east-1a.freetool.online and stored on the processing host only for as long as the conversion needs it; the source and the generated output are removed shortly after the command finishes (the retention window is the value the service emits as web.minsToDelFile at page load). The session is anonymous - no signup or sign-in is required. MP4 to GIF without an install

Turning a short MP4 clip into a shareable GIF runs the same way as any other FFmpeg job on this page: upload the source, paste a command into the parameters input, and the processing service writes the GIF you download. A practical command for a five-second loop is -i input.mp4 -ss 0 -t 5 -vf "fps=12,scale=480:-1:flags=lanczos" output.gif - the -ss and -t pair pick the start and length, the fps filter caps the frame rate (12 keeps the file small without looking jerky), and scale=480:-1 resizes the width to 480 px while preserving the aspect ratio. Choosing between this page, a Docker FFmpeg, and the in-browser converter

For a complex pipeline you would normally run locally, this hosted route skips the setup tax of pulling and warming an FFmpeg Docker container and mounting the file - you paste the same flags FFmpeg and FFprobe accept on a command line into the parameters input, and the processing service runs the binary for you. The two binaries answer different questions, so pick the dropdown first: FFmpeg re-encodes and returns a download, while FFprobe is read-only and reports codec, bitrate, and stream metadata as text rather than a downloadable file. Where the file fits comfortably in browser memory and you only need a lighter conversion, the in-browser HD Video Converter runs FFmpeg.wasm with nothing leaving your device, whereas this page uploads over HTTPS and removes the source and output shortly after the command finishes. A quick way to decide: arbitrary or chained flags belong here; a single MP4-to-WebM swap on a small clip is faster in the browser tool; and a multi-hour batch or hardware-accelerated encode is still a job for FFmpeg installed locally.
