Frequently asked questions

What is the difference between a dead pixel and a stuck pixel?

A dead pixel is permanently off - its transistor has failed and the pixel shows black on every color pattern. A stuck pixel is locked to one colour (red, green, blue, or a combination) because one or more sub-pixels are permanently powered. Stuck pixels are often recoverable by running a rapid colour-cycling animation for 10-15 minutes; dead pixels are not - only a panel replacement fixes them.

How do I know which ISO warranty class my monitor uses?

Check the product spec sheet or the support page on the manufacturer's site. Premium panels (Apple displays, Dell UltraSharp) typically apply Class I (zero dead pixels allowed). Mid-range monitors follow ISO 9241-307 Class II (up to 2 dead pixels per million before warranty applies). Budget panels may use Class III (up to 5). If the spec sheet does not state the class, call support before the return window closes.

Does the LCD test work for detecting dead pixels on phones and tablets?

Yes. The browser color sweep runs on any device with a web browser. On phones, switch to landscape orientation, turn brightness to maximum, disable Night Shift or blue-light filter, and step through each color full-screen. Dead and stuck pixels are harder to see on high-density phone panels - hold the phone at arm's length and look for any point that does not match the surrounding color.

How long does a dead pixel test take?

A basic five-color sweep (black, white, red, green, blue) takes about 2-3 minutes at normal viewing distance. If you want to test for image retention on OLED panels, leave each solid color on screen for 2-3 minutes before switching - that extends the test to around 15 minutes total. For a brand-new monitor, run the full 15-minute test before the return window closes.
