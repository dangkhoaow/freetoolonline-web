Batas file 4 GiB FAT32 - kenapa video besar gagal di USB

FAT32 masih jadi format bawaan di banyak USB stick dan kartu SD, dan menolak setiap file lebih besar dari 4 GiB kurang 1 byte (4,294,967,295 byte). Ruang kosong di stick tidak membantu - video 4.7 GB, image disk, atau installer game tetap gagal disalin meski drive terlihat kosong. Solusi andal adalah format ulang ke exFAT atau NTFS, atau pecah payload jadi bagian lebih kecil dengan alat ZIP file.

Last reviewed: 2026-07-18

Published July 18, 2026

Apa yang terjadi

Tabel perbandingan sistem file Microsoft mencantumkan ukuran file maksimal FAT32 sebagai 4 GiB. Plafon itu datang dari field panjang file 32-bit di entri direktori FAT, jadi ukuran legal terbesar adalah 2^32 - 1 byte. Volume bisa jauh lebih besar; plafon per-file terpisah dari ruang kosong yang dilaporkan stick. exFAT dan NTFS keduanya mengiklankan ukuran file teoritis maksimal 2^64 - 1 byte di tabel yang sama, jadi format ulang stick yang sama biasanya membuat salinan berhasil.

Dicek terhadap Microsoft Learn (File System Functionality Comparison) dan dokumentasi keluarga FAT pada 18 July 2026:

Sistem fileUkuran 1 file maksimalPemakaian USB tipikal
FAT324 GiB - 1 byteDefault di banyak USB murah dan kartu kamera
exFAT2^64 - 1 byte (praktis tanpa batas untuk media)USB / SD file besar lintas platform
NTFS2^64 - 1 byteDrive eksternal Windows

Batas ini ada sejak desain FAT32 di pertengahan 1990-an. Masih menusuk di 2026 karena retailer tetap mengirim USB berformat FAT32 demi kompatibilitas maksimal, sementara file sehari-hari - video 4K, ISO Windows, installer game - rutin melebihi 4 GiB.


Kenapa ini penting untuk file sehari-hari

Pesan error sering bilang stick penuh padahal belum. OS menolak satu file terlalu besar, bukan total ruang kosong. Ketidakcocokan itu muncul saat memindah dump kamera, menyalin ISO game untuk teman, atau menyimpan rekaman layar panjang langsung ke USB. Perangkat yang hanya membaca FAT32 (beberapa TV, stereo mobil, kamera lama) tidak bisa membaca stick exFAT atau NTFS tanpa format ulang, jadi pilihan format adalah trade-off antara dukungan file besar dan kompatibilitas putar terluas.


Apa yang dilakukan dengan file Anda sekarang

Jika salinan gagal sekitar tanda 4 GiB pada stick yang masih punya ruang kosong, opsi berikut menutup kasus umum:

  • Pecah payload jadi bagian ZIP di bawah 4 GiB. Alat ZIP file bisa mengemas folder besar jadi archive yang bisa dicek ukurannya sebelum salin USB.
  • Ekstrak archive di drive penuh dulu. Alat unzip file dan panduan ekstrak ZIP RAR 7Z menjaga langkah expand jauh dari stick terbatas.
  • Format ulang stick ke exFAT bila setiap perangkat di jalur mendukungnya. Itu menghapus plafon 4 GiB sambil tetap dipakai di Windows, macOS terkini, dan banyak kamera.
  • Pakai NTFS bila stick hanya untuk workflow Windows. Headroom file besar yang sama; dukungan lebih lemah di TV terkunci dan beberapa konsol.
