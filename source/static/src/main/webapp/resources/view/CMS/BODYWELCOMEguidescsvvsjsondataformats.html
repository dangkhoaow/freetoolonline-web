There is a middle ground worth keeping in mind when neither format alone fits. JSON Lines, also written NDJSON, puts one JSON object on each line of a text file, so every record keeps JSON's typing and nesting while the file still streams a line at a time the way CSV does. A ten-gigabyte export then processes one record at a time instead of forcing the whole document into memory, which is what a single top-level JSON array would demand. Most log tools, stream processors, and data warehouses read it natively.
