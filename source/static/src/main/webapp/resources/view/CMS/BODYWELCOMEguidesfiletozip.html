If the goal is to zip a folder of many files rather than one file, the same zip-file creator handles both - drop the folder into the upload panel and the tool preserves the directory tree inside the resulting archive. The reader-task framing on this page is "one file in, one .zip out"; the sibling guide at /guides/folder-to-zip.html covers the folder-input variant of the same workflow, and both routes end at the same creator. Pre-quote the recipient the input format rather than the archive size for single media files - the size cut is rarely the reason to zip one already-compressed file.
