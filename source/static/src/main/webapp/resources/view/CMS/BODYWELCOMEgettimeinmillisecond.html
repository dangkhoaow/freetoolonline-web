Privacy and local processing

Every conversion on this page runs entirely inside the browser via Date.getTime(): the click handler reads the picker and adds the millisecond fragment, so nothing about the chosen datetime is uploaded to a server, no API call is made, and closing the tab is enough to drop the working value. That keeps it useful when the timestamp originates from a private incident log, a sensitive trace, or a customer-facing event you would rather not transit a remote service.

The format-locked picker accepts the canonical MM/dd/yyyy HH:mm:ss.l shape only, so a half-typed datetime stays inert until the input is complete, and once the millisecond number appears in the result row it is ready to copy straight into a log line, an API payload, or a test fixture without retyping. The browser's local timezone is the implicit reference - when the source event happened elsewhere, adjust the input first or paste the value into the sibling milliseconds-to-date converter to read back the calendar form and verify the conversion direction matches the audit window you are documenting.
