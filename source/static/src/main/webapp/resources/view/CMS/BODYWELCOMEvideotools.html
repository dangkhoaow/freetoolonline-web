Picking a codec before you pick a container

Container choice (MP4, MOV, WebM) follows codec choice, not the other way around. H.264 remains the safe default for web embeds because every browser and every social platform decodes it without a plug-in; H.265 ships files roughly 40% smaller for the same visual quality but needs end-to-end H.265 support in the playback chain to actually open. VP9 is the open-standards option that decodes natively in Chrome, Firefox, and Edge, and AV1 is best kept for archival until decoder support catches up across older devices. Changing the container alone does not re-encode the video - MP4 typically wraps H.264 or H.265, WebM wraps VP9 or AV1, MOV wraps Apple's QuickTime - so a `.mov`-to-`.mp4` rename may need a full transcode depending on the source codec.

Converting an MP4 clip into a GIF in the browser

When the reader task is an MP4-to-GIF conversion - turning a short MP4 into a shareable looping GIF without an account or a desktop install - FFmpeg Online is the tool on this hub that handles it: the same command-line filter chain that desktop FFmpeg accepts runs inside the page, so an `ffmpeg -i input.mp4 -vf "fps=15,scale=480:-1" -loop 0 output.gif` style invocation produces a GIF directly from the source clip without any upload to a third-party server. For a clip that is already a folder of stills rather than an MP4, GIF Maker assembles the frames into a GIF at the per-frame duration you pick.
