Why some PDFs extract as blank: text layer vs scanned image

A PDF can look identical on screen yet produce either a full paragraph of text or an empty result. PDFs exported from Word, Pages, Google Docs, and design tools carry a text layer the extractor reads directly - preserving words, line breaks, and reading order. Scanned PDFs (receipts photographed with a phone, contracts fed through a flatbed) store each page as a pixel image; to a text extractor those pages are blank. If you get an empty result, try to select a word in the PDF with your cursor: if nothing highlights, there is no text layer to extract.


When to reach for OCR instead

  • Scanned paper documents. Any PDF that originated as a photo or flatbed scan needs OCR (optical character recognition) first. Run the file through a dedicated OCR tool (Adobe Acrobat, Tesseract, ABBYY FineReader), then return here for the text pass.
  • Hybrid PDFs. Some archives wrap a scanned page inside a text-layer container. If the extraction returns only page numbers or headers, part of the file was not OCR'd.
  • Handwritten notes. OCR still struggles with free-form handwriting. A handwriting-specific model such as Google Document AI will outperform generic OCR.
  • Tables and forms. A plain text extract loses rows, columns, and checkbox state. For structured form data, prefer a form-aware PDF parser.

Preserving structure (headings, lists, table rows)

Plain-text output is lossy by design. Most PDF text layers encode absolute glyph positions rather than logical paragraphs, so multi-column layouts, sidebars, and footnotes occasionally come out interleaved. Use this tool for search, cut-and-paste, or feeding text to another program. If you need headings and lists to survive, export from the original editor as HTML, or use a structure-aware library (pdfplumber, pdf.js).


Privacy

Uploads travel over HTTPS to the conversion service; both the source PDF and extracted text are removed after a short retention window. The text you download is the only persistent copy. For contracts and internal reports, save the result promptly and clear the file from any shared device once done.

Working with a PDF that won't open for text extraction? Run it through Remove PDF Password first. To keep paragraph and list markup instead of plain text, use the PDF to HTML converter. When you need each page as a standalone image rather than its text, use PDF to Images. The three converters (text / HTML / images) each extract a different slice of the same source document.

Putting the extracted text to use

Once you have the plain text, it serves jobs the PDF itself cannot. For a searchable archive, run each PDF through the converter and save its .txt beside the original; your file search (or a command-line tool like grep) can then find a phrase across hundreds of documents that a stack of PDFs would hide. For reuse, the extracted text drops cleanly into a form field, a chat message, or an email, where a copied PDF region would paste as a flat image and lose its selectable words. When you only need a single passage - a clause from a contract or a definition from a manual - taking it as text removes the retyping step and the typos that come with it. One practical habit: give each .txt the same name as its source PDF, so a phrase you find months later points straight back to the file it came from.
