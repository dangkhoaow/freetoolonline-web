Apa yang sebenarnya dimaksud dengan "dekripsi MD5"

MD5 memetakan input berukuran apapun ke output 128 bit. Operasi ini dirancang sebagai fungsi perangkap satu arah: dari output, tidak ada jalur aljabar kembali ke input. Tidak ada kunci yang ada, tidak ada tombol balik. Yang ditawarkan oleh situs yang disebut "dekripsi MD5" adalah pencarian kamus yang telah dihitung sebelumnya. Jika string pernah di-hash dan disimpan sebelumnya, teks aslinya muncul. Jika tidak - input unik, panjang, atau acak apapun - pencarian tidak mengembalikan apa-apa.

Pertanyaan yang sering diajukan

Apakah ada dekripsi MD5 sejati yang dapat membalikkan hash apapun?

Tidak. MD5 adalah fungsi satu arah berdasarkan desain. Tidak ada invers matematika, tidak ada kunci, dan tidak ada algoritma yang dapat memulihkan input sembarang dari outputnya. Alat yang disebut dekripsi MD5 melakukan pencarian kamus - mengembalikan hasil hanya jika input pernah di-hash dan disimpan sebelumnya.

Apa perbedaan antara MD5 dan SHA-256 untuk checksum?

Keduanya menghasilkan sidik jari lebar tetap untuk input apapun. MD5 menghasilkan 32 karakter hex; SHA-256 menghasilkan 64. Untuk penggunaan non-adversarial seperti kunci cache, keduanya cepat dan andal. Untuk pemeriksaan integritas, SHA-256 adalah pilihan yang tepat - MD5 memiliki serangan tabrakan yang diketahui.

Mengapa pencarian MD5 tidak mengembalikan apa-apa untuk input saya?

Pencarian memeriksa kamus yang telah dihitung sebelumnya dari string yang pernah di-hash. Jika input Anda belum pernah di-hash dan disimpan, kamus tidak memiliki catatan. Kata-kata umum dan kata sandi populer sudah tercakup dengan baik; string unik, panjang, atau acak hampir tidak pernah muncul.
