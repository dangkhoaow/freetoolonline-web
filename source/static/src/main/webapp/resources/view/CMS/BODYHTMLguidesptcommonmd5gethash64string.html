Hash MD5 no navegador - o que chamadas como `common::md5::gethash64string` realmente fazem

Se voce pesquisou por `common::md5::gethash64string` ou um nome similar de metodo no estilo framework, a operacao por baixo e o hash MD5 padrao - a mesma funcao de hash unidirecional que o gerador de hash MD5 deste site executa em dois paineis: Text to MD5 transforma qualquer entrada em uma saida hex de 32 caracteres; MD5 to Text consulta a entrada contra um cache de strings previamente armazenadas. O cache e finito - palavras comuns e frases curtas batem; strings unicas ou longas nao estarao no cache.

MD5 e uma funcao de hash unidirecional: o painel de busca reversa funciona verificando um dicionario de strings que ja foram inseridas no hasher antes, e nao invertendo matematicamente o hash. Para senhas, nunca confie em MD5 - ataques de colisao existem desde 2004 e a recomendacao moderna e bcrypt, scrypt ou Argon2. Para checksums, impressoes digitais rapidas de arquivos ou identificadores entre sistemas (o caso de uso que helpers de framework como `common::md5::gethash64string` tipicamente embrulham), MD5 continua conveniente e ubiquo. Uma saida "MD5" de 64 caracteres geralmente significa que o framework concatenou duas chamadas MD5 ou pareou MD5 com uma segunda string hex; o MD5 padrao em si sempre tem 32 caracteres hex.
