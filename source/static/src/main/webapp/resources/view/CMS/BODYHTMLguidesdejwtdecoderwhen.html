Jwt Decoder Wann Verwenden

Last reviewed: 2026-07-14

JWT Decoder lohnt sich zu oeffnen, wenn Sie lesen muessen, was ein JSON Web Token tatsaechlich enthaelt, bevor Sie ihm in Code, einem Bugreport oder einem API-Aufruf vertrauen. Diese Seite zeigt, wann das passt und wann nicht.


Wann es passt

Verwenden Sie es, um zu pruefen, welche Claims ein Token traegt, zu bestaetigen, ob sein exp-Zeitstempel bereits abgelaufen ist, oder zu verifizieren, dass eine Zeichenkette wirklich ein JWT ist - drei durch Punkte getrennte Teile - statt eines aehnlich aussehenden Base64-Blobs.


Was zu erwarten ist

Das Dekodieren laeuft lokal in diesem Browser-Tab - kein Upload und kein Konto erforderlich. Header und Payload werden als lesbares JSON angezeigt; die Signature wird als Rohtext angezeigt, aber nicht verifiziert, da das den geheimen oder oeffentlichen Schluessel des Ausstellers erfordert, der diesen Browser nie erreicht.

Nichts, was Sie einfuegen, wird gespeichert; ein Neuladen der Seite leert das Feld.
