Reach for the Developer Tools hub when you need to parse JSON, minify or unminify CSS and JavaScript, diff two text files, or hash a string with MD5 - all thirteen utilities run in your browser with no installs or accounts required. Pick the right tool: JSON Parser for malformed payloads; CSS / JS Minifier for a smaller production file; CSS / JS UnMinifier when you receive minified code and need to read it; Text Diff for side-by-side comparison; MD5 Converter for fingerprinting and integrity checks. Pasted code, hash inputs, and diff text process in-page and are discarded when the tab closes.

Production-readiness work often pairs two child routes. Paste a JavaScript or CSS file into the Minifier, copy the shrunken output into the build artefact, and keep the original on disk in case a stack trace needs readable line numbers; the UnMinifier handles the reverse for triage. For release verification, pair Text Diff (highlights added, removed, and changed lines between two config revisions) with the MD5 Converter (confirms two files are byte-identical by hash). The MD5 Converter is a checksum-fingerprint utility - use it for integrity checks rather than password storage; for cryptographic password work, reach for SHA-256, bcrypt, or Argon2 instead.

Alongside the parse / minify / diff / hash set, the hub also lists a CSS Gradient Generator that turns picked colors into a reusable gradient snippet for a stylesheet, and a Text-to-HTML Editor that converts pasted plain text into clean HTML markup for a CMS or blog template. Both run on pasted input in the page itself and discard it when the tab closes.

Pretty-print and validate JSON in the browser

The JSON Parser child route on this hub doubles as a free in-browser JSON formatter: paste a payload that came off an API response, a config file, or a log line, and the right pane renders it as an expandable tree with the syntax errors marked at the failing position. Nothing leaves the page - the parse runs in JavaScript and the input is discarded when the tab closes, so the workflow is safe for payloads that carry tokens or customer data the developer would not normally paste into a third-party site.
