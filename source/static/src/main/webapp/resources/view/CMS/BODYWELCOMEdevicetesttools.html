This hub is the starting point for four browser-based hardware checks - camera, microphone, keyboard, and screen. Most readers land here right before something depends on the gear: a video call, a recording, or a warranty return on a new or borrowed laptop. Pick the input you need to verify from the list above; each test opens on its own page and runs with no install and no sign-in.

If a test shows no signal, the cause is far more often a permission state than a broken device - so the walkthrough below leads with a permission-first checklist (the address-bar prompt, an HTTPS origin, the OS privacy panel) before you suspect the hardware. Every test on the four child routes runs entirely in the browser: the captured frames and audio are processed in JavaScript and discarded the moment the tab closes, with no account, no install, and no backend retention at any step.

A common reason people reach this hub is a free microphone test before a call - a quick way to confirm a headset or built-in mic is actually picked up, not just plugged in. The Microphone Test page handles that exact check: it reads audio input and the browser permission that most often silences it, so you learn whether the system can hear the mic before you trust it in a meeting or a recording.
