HTML Entity Encoder & Decoder - Cuándo Usar

Last reviewed: 2026-07-23

Usa el HTML Entity Encoder & Decoder siempre que un texto deba mostrarse correctamente dentro de HTML en lugar de interpretarse como marcado - un campo de comentarios, un fragmento de código en una publicación, o cualquier lugar donde texto escrito por el usuario llegue a la página.


Cuándo tiene sentido

Úsalo antes de pegar texto escrito por el usuario o no confiable en HTML puro, para que caracteres como &amp;, &lt; y &gt; se muestren como texto literal en lugar de romper el marcado. También funciona al revés: pega un fragmento de HTML que ya contiene entidades y decodifícalo de vuelta a texto legible.


Cuándo no tiene sentido

Esta herramienta escapa solo cinco caracteres - no elimina etiquetas ni scripts. Si el objetivo es eliminar marcado peligroso en lugar de mostrarlo como texto, usa un sanitizador de HTML dedicado. Tampoco es la herramienta adecuada para codificar datos binarios o archivos completos para transporte - consulta Base64 Encoder & Decoder para eso.


Qué esperar

La dirección se detecta automáticamente a partir de tu texto, sin un interruptor manual entre codificar y decodificar. Todo se ejecuta localmente en esta pestaña del navegador - sin subir nada, sin cuenta, y nada de lo que escribes se guarda al salir de la página.
