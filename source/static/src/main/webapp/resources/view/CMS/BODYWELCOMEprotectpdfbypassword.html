User password vs owner password: what each one actually restricts

PDF security has two password concepts and they do different jobs. A user password (open password) blocks opening the file at all; the reader must enter it before seeing any content. An owner password (permissions password) can restrict actions like printing or copying after the file is open. This tool sets the open password only - the lock that requires a password before any page renders. It does not set an owner / permissions password.

Signing a PDF that already has an owner password? That is the restriction that blocks most signature tools, not the open password this tool applies. Remove it first with Remove PDF Password, then sign - see How to sign a PDF after removing its password for the order of operations.


Choosing a PDF password that resists common attacks

Target twelve characters or more, mixing cases, digits, and at least one punctuation mark. Avoid dictionary words, dates tied to the document, and strings derived from the filename. A passphrase of four unrelated words is the strongest memorable option. For archival documents, consider a second layer - a ZIP password blocks the byte-level file even before the PDF viewer runs. The encryption envelope is industry-standard AES; the breakable link is the human-typed passphrase in front of it.
