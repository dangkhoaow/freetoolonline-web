So Pruefen Sie die Kameraqualitaet Ihres Telefons

30-Sekunden-Antwort. Fuenf Tests erfassen fast jedes Kameraproblem: bestaetigen Sie, dass die Live-Vorschau sich oeffnet (Kamera waehlbar und Browser hat Berechtigung), pruefen Sie die Aufloesung gegen das Datenblatt, fokussieren Sie auf Text auf Armlaenge und pruefen Sie die Schaerfe, machen Sie ein Foto bei hellem und schwachem Licht und pruefen Sie sichtbare Flecken auf der Linse.

Test 1 - die Kamera oeffnet sich

Der erste Fehlerfall ist "Kamera erscheint nicht im Browser". Zwei haeufige Ursachen: der Browser hat keine Kameraberechtigung, oder eine andere App haelt die Kamera. Fuer eine schrittweise Anleitung zur Freigabe der Berechtigung, siehe Kameratest-Berechtigung blockiert - so erlauben.

Test 2 - Aufloesung passt zur Spezifikation

Telefonkamera-Spezifikationen werden in Megapixeln beworben. Ein 12-MP-Sensor produziert 4032x3024-Fotos. Die Testseite zeigt die tatsaechliche Aufloesung, die der Browser erhaelt.

Test 3 - Fokus auf Text

Halten Sie gedruckten Text auf Armlaenge. Tippen Sie zum Fokussieren. Der Text sollte in 0,5 Sekunden scharf werden. Langsamer Fokus oder weiche Vorschau weisen auf defekten Autofokus oder verschmierte Linse hin.

Fuer eine vollstaendige Vor-Anruf-Routine siehe Geraete-Test-Checkliste fuer Remote-Arbeit. Fuer eine schnelle Pass/Fail-Uebersicht ohne einzelne Testschritte, siehe Kameracheck.

Hardware-Fehler vs. Software-Fehler - Anzeichen erkennen

Die Unterscheidung zwischen Hardware und Software ist die wichtigste Frage nach allen fuenf Tests. Ein fester Fleck in derselben Ecke jedes Fotos, unabhaengig davon, wie man das Telefon haelt, befindet sich im Kameramodul - ein Garantiefall, keine Reinigungsaufgabe. Ein Autofokus, der bei jeder Oberflaeche in jeder Lichtsituation versagt, deutet auf einen defekten Motor oder ein beschaedigtes Linsenelement hin. Rauschen, das nach einem OS-Update verschwunden ist, oder ein Fokusfehler, der nach einem App-Neustart behoben wurde, zeigt ein Software-Problem. Pixel-Binning ist eine haeufige Fehlerquelle: Ein 48-MP-Telefon liefert ueber die Browser-Kamera-API in der Regel 12 MP (4000x3000), weil vier Pixel zu einem zusammengefasst werden - korrektes Verhalten, kein Defekt.
