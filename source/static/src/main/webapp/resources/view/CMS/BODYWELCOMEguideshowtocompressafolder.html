Where the size win actually lands: file-type compression ratios

Plain-text formats - log files, CSV exports, JSON payloads, and source code - compress to 20-40% of their original size inside a ZIP, because the DEFLATE algorithm finds long stretches of repeated bytes to eliminate. A 1 MB server log typically produces a 100-200 KB archive. Office documents (.docx, .xlsx, .pptx) already bundle their content as a ZIP container internally, so the outer ZIP wrapper captures only the metadata overhead - expect 5-15% size reduction. PDF files land somewhere between: uncompressed PDFs (text-heavy technical documents) compress well, while PDFs already built with internal compression see only 5-20% reduction. Photos in JPG, HEIC, or WebP format come pre-compressed by the camera or encoder, so a ZIP typically produces 0-3% size change - a 200 MB folder of phone photos will land at roughly 198-206 MB in the archive, not 100 MB. When a folder mixes compressible text files alongside photos and videos, the overall ratio usually falls in the 15-40% range depending on how much of the byte count is text versus media.
