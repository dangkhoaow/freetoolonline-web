# SEO OPERATING SYSTEM FOR TOOL-BASED SITE

(freetoolonline.com – GitHub Pages compatible)

---

# 🧠 PART 1 — CORE PHILOSOPHY

## 1.1 SYSTEM THINKING

We are NOT optimizing pages individually.

We are building:
→ A network of tools
→ Grouped by intent clusters
→ Connected via internal links
→ Reinforced by hub pages

---

## 1.2 GOLDEN RULES

1. DO NOT break what is already ranking
2. DO NOT change URLs
3. DO NOT rewrite entire content
4. DO improve structure, clarity, and linking
5. DO scale only proven patterns

---

## 1.3 SUCCESS MODEL

Traffic growth will come from:

* Better clustering (topical authority)
* Reduced cannibalization
* Increased CTR
* Higher coverage (more queries per page)

---

# 🧱 PART 2 — SITE ARCHITECTURE MODEL

## 2.1 STRUCTURE

Flat URL stays the same.

Logical structure is created via linking:

Hub → Tool → Related tools

---

## 2.2 CLUSTER DEFINITION

### ZIP CLUSTER

Hub: /zip-tools.html
Pages:

* /remove-zip-password.html
* /unzip-file.html

---

### IMAGE CLUSTER

Hub: /image-tools.html
Pages:

* /compress-image.html
* /resize-image.html
* /crop-image.html
* /heic-to-jpg.html

---

### DEV CLUSTER

Hub: /dev-tools.html
Pages:

* /md5-converter.html
* /base64.html

---

## 2.3 HUB PAGE ROLE

* Entry page for category
* Distribute internal links
* Explain use cases
* NOT compete with tool pages

---

# 🔁 PART 3 — EXECUTION PHASES

## PHASE 1 — FOUNDATION (SAFE)

Goal: Improve structure without risk

Actions:

* Create hub pages
* Add related tools block
* Add breadcrumb
* Add internal linking

NO content rewrite

---

## PHASE 2 — ENHANCEMENT (LOW RISK)

Goal: Improve ranking quality signals

Actions:

* Add short explanations
* Add use cases
* Add trust signals
* Improve titles (CTR)

---

## PHASE 3 — OPTIMIZATION (MEDIUM)

Goal: Reduce cannibalization

Actions:

* Identify overlapping pages
* Merge content into strongest page
* De-emphasize weaker pages

---

## PHASE 4 — SCALE

Goal: Expand winning patterns

Actions:

* Create new tools
* Expand clusters
* Add new hub pages

---

# 🧩 PART 4 — PAGE TEMPLATE MODEL

Each page should evolve toward:

---

## STRUCTURE

1. Tool UI (top)
2. Short description (2–3 lines)
3. How it works (simple)
4. Use cases
5. Related tools (internal links)
6. FAQ (optional)

---

## IMPORTANT

Content is SUPPORTING, not primary.

Tool must remain dominant.

---

# 🔗 PART 5 — INTERNAL LINKING SYSTEM

## 5.1 RULES

Each page must link to:

* 1 hub page
* 2–4 related tools

---

## 5.2 LINK TYPES

### Parent:

→ category hub

### Sibling:

→ similar tools

### Lateral:

→ different cluster (optional)

---

# ⚠️ PART 6 — CANNIBALIZATION CONTROL

## RULE:

1 intent = 1 page

---

## DETECTION:

If queries overlap:

* compress zip
* zip compressor
* reduce zip size

→ must map to ONE page

---

## ACTION:

* Keep strongest page
* Merge content
* Avoid duplicate tools

---

# 🎯 PART 7 — CTR OPTIMIZATION FRAMEWORK

## TITLE FORMULA

[Action] + [File Type] + [Outcome] + [Speed/Free]

---

## EXAMPLES

Good:
Compress ZIP File Online (Reduce Size Fast & Free)

Bad:
ZIP Tool

---

## META DESCRIPTION

Focus:

* clarity
* benefit
* speed
* privacy

# 🤖 PART 9 — CURSOR PROMPT SYSTEM

Use these prompts inside Cursor.

---

## PROMPT 1 — ADD INTERNAL LINKING

"Add a 'Related Tools' section to this page.
Include 3 relevant tools from the same cluster.
Also add a link to the hub page.
Do NOT modify existing content."

---

## PROMPT 2 — ADD SUPPORTING CONTENT

"Add a short explanation (max 3 paragraphs) below the tool.
Include:

* what the tool does
* 2 use cases
* 1 limitation
  Keep it concise and non-generic.
  Do NOT rewrite existing text."

---

## PROMPT 3 — CREATE HUB PAGE

"Create a hub page for [cluster name].
Include:

* title
* intro paragraph
* list of tools with short descriptions
* internal links
  Do NOT over-optimize SEO text."

---

## PROMPT 4 — CTR IMPROVEMENT

"Rewrite the title and meta description to improve CTR.
Focus on:

* action
* outcome
* speed
* simplicity
  Do NOT make it spammy."

---

## PROMPT 5 — CANNIBALIZATION FIX

"Compare these pages:
[list URLs]

Identify overlap in intent.
Recommend:

* which page should be primary
* what content should be merged
  Do NOT suggest URL changes."

---

## PROMPT 6 — ADD FAQ

"Add 3–5 short FAQ items related to this tool.
Focus on real user concerns.
Keep answers concise."
