### 6. KEY ISSUES & ROOT CAUSES

#### Issue #1: Indexing Crisis - Only 31% Coverage ⚠️ **SEVERITY: CRITICAL**

**Root Cause Analysis:**
1. **Static Site Generator Canonical Issues** - `export-site.mjs` may be:
   - Generating multiple variants of each page (www, non-www, trailing slash)
   - Adding canonical tags pointing to wrong URLs
   - Creating redirect chains

2. **URL Parameter Duplication** - GSC shows some pages with `?utm_source=internal&utm_medium=leftmenu` tracked
   - This suggests tracking parameters are creating duplicate URLs
   - Google should ignore them, but older sites sometimes have issues

3. **Staging vs Production Bleed** - If staging URLs are being indexed:
   - Staging site being crawled (lowering coverage ratio)
   - Or staging canonicals pointing to production (correct) but creating duplicate content

**Evidence from CLAUDE.md:**
```javascript
export const STAGING = false;  // Confirms production mode
export const CANONICAL_ORIGIN = 'https://freetoolonline.com';  // Correct
```

**But also:**
```
| `BASE_PATH` | `` | Subpath prefix (staging only) |
| `STAGING` | - | Set to `true` for noindex/staging banner |
```

If `BASE_PATH` is accidentally set, it could create URL duplicates.

#### Issue #2: Mobile Traffic Underperformance ⚠️ **SEVERITY: HIGH**

**Symptom:** Desktop 79.7% of traffic, Mobile 19.9%, Tablet 0.4%

**This is backwards from 2026 standards.** Typical ratio: Mobile 55-65%, Desktop 35-45%

**Root Causes:**
1. **Mobile CTR is HIGHER** (6.66% vs 4.27%) - but mobile traffic is LOWER
   - Suggests mobile pages rank worse (higher position required for clicks)
   - Or: Mobile snippet is less appealing (title/description truncation?)

2. **Possible Mobile-Specific Issue:**

#### Issue #3: Hub Pages Invisible in SERPs ⚠️ **SEVERITY: HIGH**

**Symptom:** 8 hub pages = 0 clicks, 0 impressions
